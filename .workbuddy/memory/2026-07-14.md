## 存储层改造：KV → Blob Storage

### 背景
用户询问如何配置 KV 空间。经查 EdgeOne 文档发现：**KV 仅支持 Edge Functions（V8 运行时），不支持 Cloud Functions（Node.js）**。项目使用的是 Cloud Functions，因此 KV 方案不可行。

### 改为 Blob Storage
使用 `@edgeone/pages-blob` SDK（Cloud Functions 专用），无需在控制台手动创建命名空间，首次调用自动创建。

### 改动内容
1. 安装 `@edgeone/pages-blob@0.0.14`（latest）
2. 修改 `cloud-functions/api/[[default]].js`：
   - 添加 Blob Storage 持久化层（延迟初始化 + try-catch 降级）
   - 投稿数据：每次变更后 `persistSubmissions()` 异步写入
   - 运行配置：PUT /api/config 后 `persistConfig()` 异步写入
   - 自定义题库：审核通过/编辑器修改后 `persistPuzzles()` 异步写入
   - 冷启动时通过 `initStorage()` 中间件从 Blob 加载已持久化的数据
3. 本地环境优雅降级：`getStore()` 失败时自动回退到纯内存模式

## 修复 404 问题

### 原因
EdgeOne 框架检测将项目识别为 "other"，无法自动检测构建输出目录。`BuildResultInfos: OutputDir:` 为空，导致静态文件未正确部署。

### 修复
在项目根目录创建 `edgeone.json`：
```json
{
  "buildCommand": "npm run build",
  "outputDirectory": "client/dist"
}
```

修复后构建日志确认：`OutputDir:client/dist`
